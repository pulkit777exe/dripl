# Important: Next.js TypeScript Configuration

## Why dripl-app uses `../../tsconfig.json` instead of `@dripl/typescript-config/tsconfig.json`

Next.js (specifically Turbopack in development mode) has issues resolving package-based `extends` paths in `tsconfig.json`. 

### The Problem
When you use:
```json
{
  "extends": "@dripl/typescript-config/tsconfig.json"
}
```

Next.js throws the error:
```
An issue occurred while parsing a tsconfig.json file.
extends: "@dripl/typescript-config/tsconfig.json" doesn't resolve correctly
```

### The Solution
Use a relative path to the root tsconfig:
```json
{
  "extends": "../../tsconfig.json"
}
```

This works because:
1. Next.js can resolve relative file paths reliably
2. The root `tsconfig.json` provides the necessary base configuration
3. The app-specific `compilerOptions` override what's needed for Next.js

### Summary
- **All packages**: Use `@dripl/typescript-config/tsconfig.json` ✅
- **ws-server app**: Use `@dripl/typescript-config/tsconfig.json` ✅  
- **dripl-app (Next.js)**: Use `../../tsconfig.json` ✅ (Next.js limitation)
- **ui package**: Use `@dripl/typescript-config/react-library.json` ✅ (needs JSX)

**Do not change the dripl-app tsconfig extends path** - it must remain as `../../tsconfig.json` for Next.js to work properly.
